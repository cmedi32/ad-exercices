# Coding Conventions and Style

The project enforces coding conventions and style through static analysis tools.

## Checkstyle
- **Configuration:** `config/checkstyle/checkstyle.xml`
- **Purpose:** Enforces a consistent coding style and adherence to best practices. The specific rules are defined in the configuration file.

## PMD
- **Configuration:** `config/pmd/pmd_hslu.xml`
- **Purpose:** A source code analyzer that finds common programming flaws like unused variables, empty catch blocks, and unnecessary object creation. The ruleset is customized for the HSLU environment.

## General
- The use of these tools indicates a commitment to code quality and maintainability.
- The project follows the standard Maven project structure, which is a convention in itself.
